How the Career Pivot Agent gets built

One page. No fluff. Read this end-to-end and you should know exactly what we're shipping, in what order, with what stack — and what we're explicitly NOT building in v1.

1. What it is (and isn't)

It is

  • A single tool at huva.io/pivot that takes a job role + CV and returns a personalized AI-pivot plan.
  • A self-contained Next.js app, deployed on Vercel, routed under huva.io via rewrite.
  • Bilingual (EN + AM) from day one. Same code, two language packs.
  • Anchored in real Anthropic Economic Index research — every diagnostic number cites a source.

It is not (in v1)

  • Not a multi-tenant SaaS. No accounts, no logins, no payments.
  • Not integrated into the existing huva.io codebase. It's its own repo, mounted via routing.
  • No save/share URLs in v1 — output lives in the browser tab. (Add in v1.5 if traffic justifies.)
  • No CV upload reading on our backend — we send the PDF straight to Claude (native support).

2. User flow · 90 seconds end-to-end

1Landhuva.io/pivot
2Role + Diagnosticinstant, no input beyond a title
3CV uploadPDF or paste
4Pivot planstreams in, ~30s
5Identity + sharePDF · LinkedIn · soft CTA

Steps 2 and 4 are both Claude API calls. Step 2 is fast (small prompt, role lookup). Step 4 is the heavy generation.

4. Data flow

┌──────────┐    role+lang    ┌──────────────┐
│  User    │ ───────────────▶│  /api/diag   │   (small Claude call · returns exposure %, tasks)
│ (browser)│ ◀───────────────│  edge fn     │
└──────────┘    SSE stream   └──────────────┘
     │
     │  CV (PDF or text) + chosen pivot + lang
     ▼
┌──────────┐                 ┌──────────────┐
│  User    │ ───────────────▶│  /api/plan   │   (full Claude call · master prompt)
│          │ ◀───────────────│  edge fn     │   (streams JSON sections back)
└──────────┘    SSE stream   └──────────────┘
                                     │
                                     ▼
                              ┌──────────────┐
                              │ Anthropic    │  Sonnet 4.6
                              │ API          │  PDF input · streaming output
                              └──────────────┘

No PII stored. CV is sent to Anthropic's API and immediately discarded. We log only anonymous funnel events (role bucket, completion, no content).

5. Master prompt — what Claude returns

One Claude call (the heavy one in step 4) returns a single structured JSON object. Frontend renders it section by section as it streams in.

{
  "diagnostic":   { "summary": str, "blurb": str, "exposure_pct": int,
                    "augmented_pct": int, "automated_pct": int, "untouched_pct": int,
                    "tasks": [ { label, tag, text } x4 ] },
  "pivots":       [ { name, gap_pct, best?, why } x5 ],
  "plan_30day":   [ { week, focus, daily_block } x4 ],
  "plan_7day":    [ { day, action } x7 ],
  "linkedin":     { "headline": str, "about": str },
  "projects":     [ { name, huva_day, what_youll_build } x3 ],
  "identity":     { name, new_role, risk }
}

The system prompt enforces output language: if user picked AM, the entire JSON returns in Armenian. We do not translate post-hoc.

Cloudflare Turnstile—$0
Domain rewrite—$0 (existing huva.io)

A pivot tool that costs $0.03 per qualified lead is, frankly, an absurd ROI compared to ad spend. The real cost is build time, not runtime.

8. Build phases

v1.0 · ship first

Core experience · 6–8h

  • Single page, EN + AM, full flow (role → diagnostic → CV → output → identity card)
  • PDF download via print CSS
  • Cloudflare Turnstile
  • Soft HuVa Academy CTA, single placement
  • Vercel deploy, huva.io/pivot rewrite
v1.5 · only if traffic earns it

Share + persistence · +2h

  • Supabase row per generation, public share URLs (/p/[id])
  • OG image preview per result (auto-generated card)
  • "Saved by 1,247 people" social proof when populated
v2.0 · later, if v1 validates

Personalization deep cuts · +6h

  • More personas (15+ role archetypes pre-tuned)
  • "Apply with this CV" — one-click ATS-friendly export
  • Email follow-up sequence (Day 7, 15, 30 nudges)
  • In-tool HuVa lesson previews on hover
future · maybe

Adjacent tools

  • Same codebase hosts the Build Plan Generator (POC #2 from the plan)
  • Shared lib/anthropic.ts, shared design system, shared i18n
  • Compounding infra: each new tool ships in 4–6h, not 12–16h

9. Risks & how we kill them

RiskKill it with
Anthropic output drifts off-brandLocked system prompt + golden-output evals run on every prompt change
PDF layout breaks on long contentPrint stylesheet tested with 3 personas; max-width and page-break-inside rules
Bots scraping the APICloudflare Turnstile + per-IP rate limit on the edge function
Cost runs away if it goes viralDaily spend cap on Anthropic API key; auto-throttle to a queue if exceeded
Armenian translation feels stiltedNative Armenian review (Ani) on the EN/AM lockup before launch

What "approved" means

If you're happy with this doc, three checkboxes ship the work:

  • v1 scope is locked (everything in column 1 above; nothing more in v1)
  • Stack is approved (Next.js + Vercel + Anthropic + Cloudflare; no Supabase in v1)
  • Build budget is 6–8h focused work, target launch within 3 days

Once approved, we move to the implementation plan: a step-by-step build sheet with file-level tasks, in order.
