How the Career Pivot Agent gets built

One plan. Read it end-to-end and you'll know exactly what we're shipping, in what order, with what stack — and what we're explicitly NOT building in v1.

1. Goal

A simple, high-value AI career-pivot tool at huva.io/pivot. User drops their role → instant diagnostic of how exposed they are to AI → uploads CV → gets a personalized 30-day pivot plan, LinkedIn rewrite, 3 portfolio projects, and a shareable identity card. One soft CTA to HuVa Academy at the end.

2. What it is (and isn't)

It is

  • A single Next.js page at huva.io/pivot, mounted via Vercel rewrite
  • Bilingual EN + AM from day one (same code, two language packs)
  • Anchored in real Anthropic Economic Index research — every diagnostic is citeable
  • One Claude API call total · the diagnostic is a static lookup, not an LLM call

It is not (in v1)

  • No accounts, no logins, no payments
  • No save / share URLs · no Supabase · no database
  • Not integrated into the existing huva.io codebase · own repo, mounted via routing
  • No backend CV parsing · PDF goes straight to Claude (native support)

3. User flow · 90 seconds end-to-end

1Landhuva.io/pivot
2Role + Diagnosticinstant · local data
3CV uploadPDF or paste
4Pivot planstreams in, ~30s
5Identity + sharePDF · LinkedIn · soft CTA

Step 2 is a local lookup with zero API cost. Step 4 is the only Claude call in the entire flow.

5. The role-risk data file (the foundation)

The diagnostic is the most important "wow" moment — and the cheapest. We pre-compute it from a static TypeScript file. Build this once, every diagnostic is instant, free, and citeable.

// lib/role-risk.ts
export const ROLE_RISK = {
  "marketing-manager": {
    exposure: 47,
    aug: 38, auto: 47, safe: 15,
    blurb: { en: "...", hy: "..." },
    tasks: [
      { tag: "auto", label: { en: "Write campaign briefs", hy: "..." }, text: { en: "Claude does this in seconds.", hy: "..." } },
      // 4 tasks per role
    ],
    cite: "Anthropic Economic Index, Feb 2026"
  },
  "software-engineer": { ... },
  "customer-service-rep": { ... },
  // ~25 roles total for v1 launch
}

Fallback for unknown roles: a generic "we don't have specific data for this role yet — here's the augment-vs-automate framing" diagnostic that still feels personal. Never shows the user a dead end.

7. Build phases

v1.0 · ship first

Core experience · 5–7 hours

  • Single page, EN + AM, full flow (role → diagnostic → CV → output → identity card)
  • ~25 roles in role-risk.ts with curated diagnostics
  • One Claude call · streaming · Zod-validated response
  • PDF download via print CSS
  • Cloudflare Turnstile
  • Soft HuVa Academy CTA, single placement at end
  • Vercel deploy · huva.io/pivot rewrite
v1.5 · only if traffic earns it

Share + persistence · +2 hours

  • Supabase row per generation · public share URLs (/p/[id])
  • Auto-generated OG image preview per result
  • "Saved by N people" social proof when populated
v2.0 · later, if v1 validates

Personalization deep cuts · +6 hours

  • Expand role-risk file to 50+ archetypes
  • "Apply with this CV" — one-click ATS-friendly export
  • Email follow-up nudges (Day 7 / 15 / 30)
  • In-tool HuVa lesson previews on hover
future · maybe

Adjacent tools on the same codebase

  • Build Plan Generator (POC #2 from the original plan) on the same Next.js app
  • Shared lib/anthropic.ts, design system, i18n
  • Each new tool ships in 4–6h, not 12–16h · compound infra

What "approved" means

Three checkboxes ship the work:

  • v1 scope is locked (everything in section 7, column 1; nothing more)
  • Stack is approved (Next.js + Vercel + Anthropic + Cloudflare; no Supabase in v1)
  • Build budget 5–7 hours focused · target launch within 2–3 days

Once approved, we move straight to the implementation plan: a step-by-step build sheet with file-level tasks, ordered for execution.
